# MailMate-informed opportunities for Tecotype

MailMate demonstrates the value and the ceiling of a durable power-user niche.
Its specificity creates passionate recommendations. Weak release freshness,
limited generic discovery, and a hard-to-explain license model constrain the
visible acquisition system.

## Transferable lessons

### Let the right user self-select

MailMate does not say “better email.” It says IMAP, macOS, keyboard control,
Markdown, advanced search, smart mailboxes, encryption, tags, and
integrations.

Tecotype should be equally specific after validation, using a customer job
rather than a technical inventory:

- “Find mail the way you remember it” with a real multilingual example; or
- a complete keyboard triage sequence for a professional inbox.

### Earn recommendations through a distinctive job

MailMate repeatedly appears in discussions where somebody asks for Markdown,
advanced search, privacy, IMAP, or keyboard control. The recommendation works
because the match is exact.

Tecotype should identify one such trigger, make the proof inspectable, and
give a small number of relevant reviewers a dependable release.

### Treat provider maintenance as product work

MailMate's founder describes provider “kill switches,” OAuth risk, and IMAP
server behavior. Tecotype supports Gmail and generic IMAP, so provider
compatibility, authorization, safe mutations, and recovery are acquisition
requirements, not background engineering.

## Staged plan

### Now: validate the niche

Interview 8–10 email-heavy professionals about a recent:

- failed or slow message retrieval;
- cross-language search incident;
- keyboard workflow;
- multi-account/provider problem; or
- offline/unreliable-network need.

Use repeated incidents, attempted workarounds, and switching behavior to select
one lead job. Do not assume “power user” is a homogeneous segment.

### Before public Mac acquisition

Tecotype has a signed Apple Silicon target and release tooling under active
development, but no public Mac release is proven. Complete:

- signed and notarized distribution;
- dependable no-account and first-run behavior;
- Gmail restricted-API readiness;
- generic IMAP setup and failure recovery;
- safe compose, sending, and synchronization;
- clear local-data and Better-search resource explanations;
- licensing, payment, cancellation, and refund behavior;
- updates and rollback planning;
- support expectations; and
- privacy-preserving source-to-activation measurement.

Avoid a “stable” label until the exact public artifact and updater path have
been verified.

### First Mac experiment

After the gate:

```text
one relevant reviewer or community question
→ one source-tagged job page
→ exact shipped demonstration
→ signed Mac evaluation
→ successful account connection
→ first lead-job completion
→ voluntary day-2 and day-7 return
→ payment
```

Measure provider/setup failure and support burden as carefully as page
conversion. A download or positive comment is not activation.

### Integration experiment

MailMate's SpamSieve, Hookmark, and BusyContacts references show the relevance
of adjacent-tool distribution. Tecotype should explore an integration only
when:

- interoperability is genuinely useful;
- both products share a validated audience;
- the integration is shipped and documented; and
- source-tagged activation can be measured.

Do not build an integration for a backlink.

### Windows plan

Windows distribution is planned and release-gated, not shipped. Prepare
Windows x64 / Windows 11 signing, packaging, credential protection, updates,
first run, and support separately. Mac evidence can inform the job and message,
but not prove the Windows acquisition channel. Publish Windows availability
only after a real signed release is verified.

## Avoid

- an unexplained beta as the default public acquisition artifact;
- a stable release that is years behind the recommended build;
- support promises that exceed capacity;
- a payment model whose practical entitlement requires several paragraphs;
- broad SEO pages disconnected from first value;
- inferring demand from advocacy without a cohort; and
- lowering Tecotype's price because a solo niche product charges less.

Tecotype's decided **€149/year or €15/month** price should be validated against
the complete professional experience, retention, provider-maintenance cost,
support burden, and refunds.
