# Airo Mail-derived opportunities for Tecotype

Scope: **marketing and distribution only**. Assume the Tecotype client works
and product delivery is handled elsewhere. Mobile is out of scope. Assume
published Tecotype supports macOS, Windows, and Linux; public campaigns must
still use only release routes verified as live at campaign time.

## 1. Segment-specific desktop landing page

### Hypothesis

One professionally specific audience will respond better to a concrete inbox
job than to a general “AI email client” position.

### Test

Create one page for a validated segment, initially founders, developers, or
consultants. Keep the structure close to Airo’s strongest pattern:

1. name the role;
2. show the mail decisions that role handles;
3. demonstrate one keyboard, calm-control, or Better search outcome;
4. state the local mailbox boundary precisely;
5. show €149/year and €15/month without a pricing puzzle; and
6. provide one source-tagged next action.

Run only one segment at a time. Compare qualified evaluation, activation,
voluntary return, payment, and renewal.

## 2. Faithful no-account demonstration

One captured Airo variant's no-sign-in live-demo link is its strongest
evaluation design, although a later direct fetch did not show it. Tecotype can
test a desktop demonstration before asking for mailbox access without treating
Airo's variant as conversion proof.

Candidate demonstrations:

- a keyboard triage sequence with visible commands;
- “train ticket to copenhagen” finding “Din togbillet til København”; or
- a quiet Split Inbox setup with an inspectable rule.

The demonstration can be a controlled marketing simulation or recorded product
walkthrough, but must stay faithful to current behavior. Track completion and
the next action by original source.

## 3. Privacy proof as distribution asset

Airo shows that privacy language needs a detailed data-flow handoff. Tecotype
has a clearer structural claim: its infrastructure never receives user
credentials or mailbox contents.

Create a concise, permanent trust page that states:

- the client connects directly to the user’s provider;
- what stays local;
- what the operating system and provider still receive;
- what optional Better search downloads and processes; and
- what marketing analytics can and cannot observe.

Use it in reviewer briefs, comparison pages, and partner listings. Do not say
mail “never leaves the device,” because it still moves through the user’s mail
provider.

## 4. Desktop discovery surfaces, not mobile stores

Airo’s stores occupy branded search despite tiny adoption. Adapt the principle
to desktop:

- platform stores when appropriate;
- package or application catalogs with durable pages;
- specialist reviewer databases; and
- permanent integration partner pages.

Each route must preserve attribution and be evaluated through activated-paid
outcomes. Do not add a listing merely for a backlink.

## 5. Honest comparison content

Airo’s iPhone comparison discloses that the author built Airo. Tecotype should
use the same candor on any alternative page:

- disclose authorship;
- identify who should choose the competitor;
- compare one real job rather than a feature spreadsheet;
- use current prices and platform states; and
- link to primary sources.

Test one page only after a buying-intent query or reviewer question is observed.

## 6. Founder community participation

The Airo Reddit response is disclosed and specific but has no visible
conversion evidence. Tecotype should participate only where the exact job is
already being discussed. Answer the question fully, disclose affiliation, and
link only when the product directly matches.

Use a unique source tag. Continue only if the route creates qualified
evaluation and activated paid users, not because a comment receives votes.

## Avoid

- Mobile acquisition or mobile feature work.
- Broad “for everyone overwhelmed by email” campaigns.
- Treating Airo’s $8 Personal or $300 Pro annual offers as price validation.
- Serving conflicting audience, price, or demo variants without controlled
  assignment and a canonical offer.
- Counting 10+ downloads or one rating as customer proof.
- Role-page or comparison-page factories before one page converts.
- Brand monitoring that does not exclude GoDaddy Airo and other collisions.
- Vague local or encrypted claims that require a security page to correct them.
- Claiming a desktop platform before its public release route is live.

## Evidence threshold

One observed Airo variant supports testing a narrow audience page and
no-account demo; the security page supports a precise trust asset. Its public
evidence does not justify scaling SEO, community, stores, or the high-price
offer. Scale requires permanent discovery and an attributable
activation-to-paid funnel.
