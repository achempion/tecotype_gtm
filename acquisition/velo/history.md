## Landing-page stages

### Initial page

The [initial landing commit](https://github.com/avihaymenahem/velo/commit/d2e6019ac53382c3c2775d280be6511e29edc885) used:

- hero: **“Email at the speed of thought”**;
- description: a free, open-source desktop email client;
- modifiers: AI-powered, keyboard-first, privacy-focused;
- proof-by-breadth: **122 features**;
- GitHub releases and repository as the actions.

It also used the wrong `valomail.app` spelling until a later same-day commit.

### Redesigned page

The [2026-02-15 redesign](https://github.com/avihaymenahem/velo/commit/6ee9eba582295073e3ee3a8c02268c63a8c2a6fa) describes itself as Linear/Stripe-inspired, typography-driven, and dark-only. It established the current core:

- “The email client you’d build for yourself”;
- free forever because open source;
- keyboard, AI, and privacy pillars;
- direct GitHub handoff.

Subsequent substantive landing-source updates added Apache licensing and new AI/tasks claims. The only landing commits after 2026-02-18 were dependency updates on February 25 and 27.

The [2026-03-04 archive](https://web.archive.org/web/20260304214115/https://velomail.app/) and [2026-03-17 archive](https://web.archive.org/web/20260317224529/https://velomail.app/) preserve the same HTML asset hashes and metadata as one another. The current page remains materially aligned with this stage.

## Release cadence

The GitHub API returned:

- 61 public release entries;
- 36 entries with assets;
- 270 assets;
- 2,725 aggregate asset requests;
- latest release on 2026-02-27.

The high early release count supports a story of visible responsiveness. It also records repeated fixes to IMAP sync, attachments, sent mail, OAuth, packaging, updates, and other core paths. Release volume by itself is neither quality nor retained use.

## What history can and cannot establish

Observed chronology supports:

- rapid public building;
- message revision after launch;
- fast multi-provider and feature expansion;
- a Reddit-centered attention event;
- persistent distribution and setup risks.

It cannot establish:

- which change increased conversion;
- whether fast feature expansion improved or harmed retention;
- unique users by release;
- acquisition cost;
- active use or churn;
- causal influence of Reddit, LinkedIn, HN, video, or articles.
